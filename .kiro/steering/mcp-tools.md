# MCPツール活用ルール

開発ワークフローにおけるMCPツール(Serena・Context7)の利用規約。コード探索・設計・実装の各フェーズで、汎用ツールより専用MCPツールを優先することで、精度の高いコード理解と最新情報に基づいた技術判断を担保する。

## Serena(セマンティックコード操作)

- 既存コードを読む・探す際は、grep/catではなく必ずSerenaのfind_symbol/search_text等を優先する
- /kiro-spec-design, /kiro-impl フェーズでは、実装対象ファイルと関連する型・テストをSerena経由で確認してから設計・実装する
- 大規模な変更やリファクタ前には、Serenaでfind_referencesし影響範囲を洗い出す
- プロジェクト未オンボーディングの場合、最初のタスク実行前に自動でオンボーディングを行う

## Context7(最新ライブラリドキュメント)

- 外部ライブラリ・フレームワークのAPIを使うコードを書く/設計する際は、必ずContext7で最新ドキュメントを確認してから実装する
- /kiro-spec-design で技術選定や外部依存を検討する際、Context7で最新バージョンの機能・非推奨APIを確認する
- バージョンが明確な場合はlibrary IDを直接指定する(例: /vercel/next.js/v15)
