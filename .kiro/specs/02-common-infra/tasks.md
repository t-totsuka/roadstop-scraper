# Implementation Plan

- [x] 1. Foundation: commonパッケージとレジューム保存先の下地整備
  - 共通基盤モジュール群を配置するcommonサブパッケージと、対応するテストパッケージの雛形を作成する
  - レジューム状態の保存先ディレクトリ(`.resume/`)を`.gitignore`へ追加し、バージョン管理対象外にする
  - 観測可能な完了条件: `import roadstop_scraper.common`が成功し、`.resume/`配下のファイルが`git status`の未追跡一覧に現れない
  - _Requirements: 4.1_

- [x] 2. Core: 共通基盤4コンポーネントの実装
- [x] 2.1 (P) 共通ロギングセットアップを実装する
  - `python_util.logging`のロガー取得機能を共通のimportパスとして再公開する
  - スクレイピングの開始・終了・失敗イベントを一貫したメッセージ形式で記録するヘルパー関数(開始/終了は`INFO`、失敗は`ERROR`)を提供する
  - 観測可能な完了条件: ロガー取得が`logging.Logger`を返し、各イベントヘルパーが想定ログレベル・target名入りのログレコードを1件ずつ出力することがテストで確認できる
  - _Requirements: 1.1, 1.2, 1.3, 1.4_
  - _Boundary: LoggingSetup_

- [x] 2.2 (P) geo-json/index.jsonの読み書き・更新機能を実装する
  - index.jsonを読み込み、不変データ構造として返す(ファイル不在時は空の初期状態を返す)
  - JSON構文エラー・構造不正(`files`が非リスト、エントリの`path`/`updated_at`欠落等)をいずれも専用例外`IndexFileCorruptedError`に正規化して送出する
  - 指定`path`のエントリを更新(未登録なら新規追加)した新しい不変データを返し、元データは変更しない
  - `updated_at`は`python_util.time_utility.now()`によるJSTのaware datetimeをISO 8601形式でシリアライズする
  - 更新後のデータをindex.jsonへ書き込んで永続化する
  - 観測可能な完了条件: 読み込み→更新→保存→再読み込みの往復でデータが一致し、破損ファイルに対して専用例外が送出されることがテストで確認できる
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5, 2.6_
  - _Boundary: IndexStore_

- [x] 2.3 (P) リクエスト頻度制御を実装する
  - 最小待機時間(秒)をコンストラクタで受け取り、呼び出し側が設定変更できるようにする(負値は`ValueError`で拒否する)
  - 初回の待機呼び出しは待機せず即座に返り、2回目以降は直前の呼び出し完了から最小待機時間以上が経過するまでブロッキング待機する
  - 観測可能な完了条件: 初回呼び出しが即座に返ること、最小待機時間未満での連続呼び出しが実際に待機すること、十分間隔が空いていれば待機しないことがテストで確認できる
  - _Requirements: 3.1, 3.2, 3.3_
  - _Boundary: RateLimiter_

- [x] 2.4 (P) レジューム状態の永続化・復元・クリア機能を実装する
  - 保存先ディレクトリの既定値を`.resume/`としてモジュール定数に固定し、キー単位でJSONファイルとして進捗状態を保存する
  - 保存済みキーの読み込みは保存内容と等価なdictを返し、未保存キーの読み込みは`None`を返す(呼び出し側の「最初から開始」判断に使う)
  - 指定キーの永続化状態を削除するクリア操作を提供する(正常完了時に呼び出される想定)
  - 観測可能な完了条件: 保存→読み込みの往復で内容が一致し、未保存キー・クリア後のキーの読み込みが`None`を返すことがテストで確認できる
  - _Requirements: 4.1, 4.2, 4.3, 4.4_
  - _Boundary: ResumeStore_

- [x] 3. Validation: 統合検証と品質ゲート
- [x] 3.1 index.json更新フローの統合検証
  - 一時ディレクトリ上の実ファイルに対して読み込み→エントリ更新→保存の一連の流れを実行する
  - 観測可能な完了条件: 生成されたJSONが`structure.md`定義のindex.jsonフォーマット(`files`配列、`path`と`updated_at`)と一致することがテストで確認できる
  - _Requirements: 2.1, 2.4, 2.5, 2.6_
